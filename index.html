Steps we’ll use during this tutorial

The steps we will use are these:

Setup

  1. A game consists of two players: Player 1 and Player 2
  2. Player 1 thinks of a secret number, normally between 1 and 50, and remembers it

Game play

  1. Player 1 asks Player 2, “What’s your guess?”
  2. Player 2 tells Player 1 a number
  1. Player 1 listens to and remembers Player 2’s guess
  2. Finally, Player 1 will say one of three things:
  1. “Too high!” if the guess is greater than the secret number
  2. “Too low!” if the guess is less than than the secret number
  3. Or, “Correct! The secret number is <the secret number>!” if the guess is equal to the secret number

Pretty simple, right? If you haven’t already done so, play the game a few times with your friend, taking turns being Player 1 and Player 2.

Playing allllll the rounds in with one click

You just created a program that you had to modify each time you wanted to simulate each round that Jason needed to get to the right answer. But, what if we want to simulate all four of his rounds very quickly?


We can do this easily in JavaScript by introducing two more important programming concepts. I know, I just said you were finished with your first game, but I think you'll like this! The two new concepts are arrays (lists of similar things you want a program to use or remember) and looping, or doing the same thing to every item in an array.


This next program shows how you can do this in JavaScript. Since you might be getting tired in this lesson, I've omitted the line numbers to make it easier for you if you want to cheat and copy-and-paste the code. 

var secretNumber = 41;
var rounds = [43, 30, 39, 41];
for(var round = 0; round < rounds.length; round = round + 1) {
 var guess = rounds[round];
 if (guess > secretNumber) {
  say("Too high!");
 } else if (guess < secretNumber) {
  say("Too low!");
 } else {
  say("Correct! The secret number is " + secretNumber + "!");
 }
}

In this short line of code you are demanding something of the computer. Basically, you are demanding that it get ready to remember something for you! Because computers aren’t as smart as people,  we need to write very explicit code o tell the the computer what do and what to remember for us.

Mission 2: Understand ReferenceError and how to work with values that are already defined in JavaScript

At this point, in your journey, you’ve seen that the Console won’t respond positively when you type gibberish into the command line prompt. Why is that?

To start with, you have to understand that a programming language is somewhat like a spoken language. If you speak English as your first language, and try to have a conversation with someone who doesn’t speak English, but don’t use words of their own language, then you’re going to have a hard time communicating!

Experiment: Imagine that you only know how to speak English and you meet someone who only speaks Spanish. Are there some things you might be able to communicate with even if you don’t know any words in each others’ languages?

So what is going on here? Well, previously you typed in numbers and mathematical operation characters like +, -, *, and /, and JavaScript evaluated those expressions to calculate a mathematical result for you. This time, you have typed what’s called a string literal value. JavaScript understands anything between a pair of double-quote characters, “…”, or a pair of single-quote characters, ‘…’, as a string literal. Each individual letter inside the string is called a character.

Aside: A Single Character is Just a One-Character-Length String

JavaScript, unlike such languages as C, Java, C#,  F#, and many others, does not recognize the difference between a single character and a string. Technically speaking, a single character enclosed within a pair of double or single quotes is just a 1-character-length string.

In these examples, we introduced typeof. This is also an operator, similar to +, -, *, and /, but unlike those it is a unary operator, because it operates on only a single value. If you’re wondering, +, -, *, and / are all binary operators, because they operate on two values to produce a result.

The purpose of the typeof operator is to return the data type of the value it operates on. The technical term for the value an operator operates on is called “operand”. It’s a weird word, but you’ll see it frequently in programming.

The number type and string type are both primitive data types that JavaScript supports. There are not too many primitive data types. Here’s another one. Type the following into the console:

  • typeof false

What the heck does boolean mean? Well, it’s named after George Boole, an English mathematician philosopher, and logician. A boolean data type, sometimes called a logical data type, can have just two values: true, or false. Unlike strings and numbers, which can have an infinite variety of values, this one is restricted to just those two. Despite only being able to have two values, this data type is extremely important all throughout JavaScript, and dozens of other programming languages.

To demonstrate this, try a few more experiments with some new binary operators that you can use with numbers:

  • 1 > 0 – Greater than comparison operator

  • 50 <= 100 – 50 – Less than or equal to comparison operator

  • 101 > (2 + 48.4) * 2 – Greater than operator

To understand what JavaScript does for each of these expressions, let’s first understand the basic idea behind a comparison operator.

First of all, a comparison operator is a binary operator. As you learned earlier this means it operates on two operands. Let’s look at the first one: 1 == 1.

A binary operator has operands on the left-hand-side, and the right-hand-side of the operator.

In our case, the value 1  appears on both the left-hand-side and the right-hand-side.

The characters == next to each other represent the equality operator. This means that JavaScript checks to determine whether what is on the left-hand-side is equal to what is on the right-hand-side. Since 1 is equal to 1, the value that is true, a boolean, or logical, value.

In the second example, 50 == 25 * 2, JavaScript first evaluates both sides of the binary expression before attempting to make the comparison. This means that this expression gets converted into:

50 == 50

And, since 50 is equal to 50, the result is also true.

Mission 5: collecting primitives into objects

Now that you understand how to assign variables, and have gotten lots of practice doing so, you might be wondering if there is an easier way of grouping related information together. The curly braces, { and }, create the object and group the values together. Notice that instead of using the = operator, like you did when defining separate variables, we use the colon, :, to initialize the individual values within the object.
